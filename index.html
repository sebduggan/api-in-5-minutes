A REST API in 5 minutes with Preside

  • Early days of the web: sites were isolated
  • Little server-side processing
  • Browsers not able to access data client-side
  • Pull/push data via FTP
  • SOAP - How many used? How many still using?
  • Thankfully, REST came along - simpler, especially when coupled with JSON
  • Now nearly every site consumes data from the outisde
  • Consumer: Twitter, etc
  • Mobile apps need a simple way to coomunicate data
  • Enterprise: link to external SaaS services, sync CRM data, raise invoices
Preside
  • Preside is very data-centric
  • Many clients require syncing of data from external systems
  • More and more, required to make data available from Preside
  • Preside has built-in REST capabilities, based on Taffy?
  • Made sense to make the process of setting up an API as easy as possible...

A pragmatic approach to REST

The Preside Data API uses four verbs:

  • GET
  • POST
  • PUT
  • DELETE

PATCH is not used

All updates use PUT

Only supplied fields are updated

Response body contains only the data

Metadata is passed in the headers

API is documented separately

No to HATEOAS Hypertext As The Engine Of Application State

Data change queues

  • By default, turned off for users
  • Enable data change queue per object, per operation
  • Here, just enable all operations, all objects
/config/Config.cfc 

settings.rest.apis[ "/data/v1" ].dataApiQueueEnabled = true;
settings.rest.apis[ "/data/v1" ].dataApiQueues = {
    default    = { pageSize=100 , atomicChanges=true  }
  , highvolume = { pageSize=1000, atomicChanges=false }
};
By default, queue page size is 1 record. Atomic - each individual change is queued in order.

Authentication

Basic authentication

  • Users manually created in admin
  • Access token generated
  • Potential for self-service for website users
    • Programmatic creation of API user
    • Business logic defines permissions
    • Show the user their access token
    • Allow them to reset their token

No authentication

WARNING!
Only use for a read-only API

/config/Config.cfc 

settings.rest.apis[ "/data/v1" ].authProvider = "";
Could use for a firewall-protected or internal application, but even so should probably use Basic auth!
